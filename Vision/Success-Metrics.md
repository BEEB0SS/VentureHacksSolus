# Success Metrics

> Part of [[Vision]]

## Hackathon Success Criteria
1. **All 5 demo flows work end-to-end** — Change Propagation, Live Bench, Team Memory, External Knowledge, Simulator Loop
2. **The "wow moment" lands** — when a KiCad chip swap automatically highlights impacted software, the audience gets it
3. **The product feels like ONE thing** — not four people's code stitched together
4. **The pitch is clear** — judges understand the vision in under 60 seconds
5. **Live demo runs without crashes** — seeded data, simulated telemetry, smooth transitions

## Key Proof Points per Demo
- **Demo A**: Swap a chip → system instantly shows what software breaks
- **Demo B**: Live telemetry streaming → anomaly detected → AI diagnoses it
- **Demo C**: Log an issue → later find it via semantic search
- **Demo D**: Ask for a component → get grounded recommendation with specs
- **Demo E**: Change a parameter → sim vs reality diverge → system explains why

## Post-Hackathon
- Feedback from robotics engineers who try the product
- Interest from potential early adopters
- Clarity on which demo flow resonated most (informs what to build next)
