# Competitive Landscape

> Part of [[Go-To-Market]]

## How Solus Differs

| Tool | What It Does | Where Solus Wins |
|------|-------------|-----------------|
| **Foxglove** | ROS visualization & debugging | Solus connects hardware + software + team knowledge; Foxglove is runtime-only |
| **rerun.io** | Multimodal data logging & viz | Solus understands cross-domain impact; rerun logs data but doesn't reason about it |
| **ROS tooling** | Middleware + CLI tools | Solus provides system-level understanding, not just topic inspection |
| **Cursor / Copilot** | AI code editor | Solus understands the robot as a system, not just the code |
| **Onshape / KiCad** | Design tools | Solus reads from these — connects design changes to code impact |

## Our Unique Position
No existing tool:
1. Understands the robot across hardware, software, and runtime simultaneously
2. Detects cross-domain change impact (PCB change → software impact)
3. Preserves and retrieves team knowledge semantically
4. Grounds AI answers in the project's actual context model

## What We're NOT Competing With
- We don't replace CAD, PCB, or code editors
- We don't replace ROS as middleware
- We're the connective tissue between all of them
